Az assembler által támogatott utasítások

 

ORG

Az ORG utasítással tudjuk az utasítás számláló értékét beállítani.

ORG $3000

A programban több ORG utasítás is elhelyezhetõ, így a program több szekcióra osztható. Az utasítás operandusa $0000 és $FFFFF közötti érték lehet vagy a .LC konstans, mely az utasítás számláló aktuális értékét tartalmazza.

Ha az utasítás elé címkét teszünk, akkor a címke értéke az utasításban megadott cím lesz.

END

Az utasítással a program végét jelöljük. Az utasítás utáni sorokat az assembler nem veszi figyelembe.

 

DB (Define Byte)

A DB utasítással egy vagy több byte-ot helyezhetünk el a programban az utasítás helyén.

DB $AB,123,$1D  ;Három byte-ot helyez el a memóriában.

szoveg  DB ’Árvíztûrõtükörfúrógép’ ; A karakterláncnak megfelelõ ASCII kódokat helyezi el a memóriában

DB ’A’,’B’,’C’ ;A három karakternek megfelelõ ASCII kódokat helyezi el a memóriában. Azonos a DB ’ABC’ utasítással.

 

DW (Define Word)

Hasonlatos a DB utasításhoz annyi különbséggel, hogy nem egy, hanem két byte-os értékeket helyez el a memóriában.

DW $ABFF,$ABCD,$0000

 

DS (Define Storage)

Az utasítással a programban tárhelyet tudunk létrehozni.

DB darabszám, érték

A darabszám operandussal tudjuk megadni, hogy az érték operandusból hány darabot szeretnénk elhelyezni a memóriában. Az érték operandus elhagyása automatikusan nullás byte értéket jelent.

 

DS 10 ;10 darab nulla értékû byte elhelyezése a memóriában. Azonos a DB 10,$00 vagy DB 10,0 utasításokkal.

DS 5,$AB ;Az $AB értékû byte-ot 5-ször helyezi el a memóriában

DS 10,’A’ ;Az ’A’ karakternek megfelelõ ASCII értéket 10-szer helyezi el a memóriában

DS 10,’ABC’ ;Az  ’ABC’ karakterláncnak megfelelõ ASCII byte sorozatot 10-szer elhelyezi a memóriában

 

EQU

Az utasítással értéket rendelhetünk címkéhez, melyet egyéb utasításokban használhatunk.

VIDEOCIM EQU $ACFF

LD (VIDEOCIM),$2B

Az EQU utasítás helyett az egyenlõség jel is használható.

 

INCBIN

Az utasítással bináris állományt ágyazhatunk be az assembly programba.

INCBIN „filenév”,[startoffset],[mennyiség]

Az utasítás elsõ operandusa a file név, melyet idézõ jelek között kötelezõ megadni. A második operandussal azt a byte offset-et adhatjuk meg, ahonnan a beágyazást kezdeni szeretnénk. Az offset értéke 0 és file hossz-1 értékig terjedhet. Ha a második operandust nem adjuk meg, akkor a teljes file beágyazásra kerül.

Az utasítás harmadik operandusa szintén opcionális. A beágyazott byteok mennyiségét tudjuk vele megadni. Ha a harmadik operandust elhagyjuk, akkor a beágyazás a második operandusban megadott byte offset-tõl a file végéig tart.

 

INCBIN „sprite.bin”,0,30 ; 30 byte beágyazása az elsõ byte-tõl kezdve

INCBIN „sprite.bin”,40 ; Az elsõ 40 byte beágyazása a sprite.bin fileból.
